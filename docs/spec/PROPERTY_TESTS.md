# Property Tests Obligation (v0.2)

This document defines the `core/obligation::property-tests` obligation, which executes deterministic, seed-recorded property tests and emits evidence artifacts.

Property tests in v0.2 are intentionally simple and deterministic: they are pure (no effects) and are driven by seeds generated by the toolchain. Failing seeds are recorded as evidence for exact reproduction.

## Configuration (`package.toml`)

Property tests are enabled by listing suite symbols under `property_tests` and including the obligation ID:

- `property_tests = ["my/pkg::properties", ...]`
- `obligations = ["core/obligation::property-tests", ...]`

Optional config:

```toml
[property]
cases_per_test = 64
```

## Suite Shape

Each suite symbol must evaluate to a map. Each entry is a property test.

Allowed shapes for each map value:

1. A callable (closure or native fn) that accepts one argument: `seed` (int).
2. A map with:
   - `:body` callable (seed -> result)
   - optional `:cases` integer (overrides default `cases_per_test`)

## Semantics

For each property test, the toolchain generates `N` cases (seeds):

- `N` is `:cases` if present, otherwise `[property].cases_per_test` if present, otherwise the v0.2 default `64`.
- Seeds are generated deterministically as 64-bit integers derived from BLAKE3 over:
  - a versioned tag
  - package name
  - suite symbol
  - test name
  - case index

Each case calls the property function with the seed, and the case passes iff the result is boolean `true`.

Any other result is a failure:
- `false` fails the case.
- a sealed protocol ERROR fails the case.
- returning an effect program fails the test (property tests must be pure in v0.2).

## Evidence Artifact

The obligation must write a report artifact:

- `:kind = "genesis/property-tests-v0.2"`
- `:package` (string)
- `:ok` (bool)
- `:config` (map): effective configuration (`:cases-per-test`)
- `:tests` (vector): one per property test:
  - `:suite` (symbol)
  - `:name` (string)
  - `:cases` (int)
  - `:ok` (bool)
  - `:seeds` (vector of int) (all generated seeds, in order)
  - `:first-failure` (map, optional):
    - `:i` (int)
    - `:seed` (int)
    - `:result` (term) (canonical CoreForm data term for debugging)
- `:errors` (vector of strings) if `:ok = false`
